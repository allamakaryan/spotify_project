# Spotify Data Ingestion & Analytics Pipeline

## Պրոյեկտի Նկարագրություն
Այս պրոյեկտը հանդիսանում է տվյալների մշակման լիարժեք ETL (Extract-Transform-Load) pipeline: Այն կարդում է Spotify-ի հանրային դատասեթը, իրականացնում է տվյալների մաքրում և տիպերի ստուգում Pandas գրադարանի միջոցով, ապա նորմալիզացված տեսքով լրացնում PostgreSQL տվյալների բազա:

## Տվյալների Աղբյուր (Dataset)
- **Աղբյուր:** Kaggle (Spotify Tracks Dataset)
- **Ծավալ:** 114,000+ սկզբնական տող, մաքրումից հետո՝ 89,740 երգ:

## Բազայի Կառուցվածք (Database Schema)
Տվյալները նորմալիզացվել են և բաժանվել 5 փոխկապակցված աղյուսակների՝
1. `genres` - Ժանրերի հավաքածու (Unique genre names)
2. `artists` - Արտիստների ունիկալ տվյալներ
3. `albums` - Ալբոմների անվանումներ
4. `tracks` - Երգերի հիմնական տվյալները և աուդիո-մետրիկաները (Fact table)
5. `track_artists` - Միջանկյալ աղյուսակ (Many-to-Many կապի ապահովման համար)
